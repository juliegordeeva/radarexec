# Промпт для Cursor: сайт RADAR EXEC

Этот промпт заменяет предыдущий общий вариант. Он собран на основе утверждённого дизайн-гайда бренда RADAR EXEC и описывает конкретный сайт, а не generic advisory-шаблон. Вставляй целиком в Cursor (Composer/Agent), желательно в новый Vite-проект.

---

## 0. Кто мы и какое ощущение должен давать сайт

RADAR EXEC — бутиковое advisory-партнёрство для собственников, CEO, C-level лидеров и топ-команд, которые сталкиваются со сложными управленческими ситуациями.

Это **не** сайт тренингов, не личная страница эксперта, не каталог спикеров и не корпоративный шаблон. Это должно ощущаться как частный стратегический офис, премиальная advisory-студия, сдержанный международный бренд — место, куда обращаются, когда ситуация чувствительная и требует взрослого подхода.

Ключевая фраза, которую должен транслировать весь сайт целиком (тон, верстка, паузы, формулировки):

> «Эти люди умеют спокойно, точно и бережно работать с задачами, где цена управленческой ошибки высока».

Центр сайта — RADAR EXEC как командное бутиковое партнёрство и curated ecosystem senior-практик. Юлия Гордеева — важная founder-level фигура и стратегический архитектор approach, но **не** личный бренд сайта.

## 1. Стек и базовая настройка

- React 18 + TypeScript + Vite
- React Router (один длинный landing с секциями + якорные ссылки в навигации; страницы практик/форматов можно сделать как `/practices/:slug` если потребуется детализация — заложить такую возможность архитектурно, но на старте достаточно одной страницы `/`)
- Tailwind CSS — все цвета, шрифты, размеры и отступы должны быть заведены как **дизайн-токены** в `tailwind.config.ts` (см. раздел 2), а не как произвольные значения в классах
- `react-helmet-async` (или аналог) для SEO-тегов
- ESLint + Prettier
- Шрифты подключаем через Google Fonts / Fontsource (см. раздел 2.2)
- Статический деплой (Vercel/Netlify), без backend; форма — заглушка с понятной точкой подключения (например, Formspree/Tally/собственный endpoint — оставить TODO-комментарий)

## 2. Дизайн-система (дизайн-токены)

### 2.1 Цвета

Заведи цвета в `tailwind.config.ts` как именованные токены (не хардкодить hex в компонентах):

```ts
colors: {
  graphite: {
    deep: '#111111',   // фон hero, RADAR Races, финальный CTA
    DEFAULT: '#151515',
    light: '#1A1A1A',
  },
  ivory: {
    DEFAULT: '#F4F0E8', // светлый фон / текст на тёмном
    soft: '#EFE8DC',
  },
  stone: {
    DEFAULT: '#9D9A93', // второстепенный текст
    light: '#B7B0A6',
  },
  taupe: {
    DEFAULT: '#3A352F', // внутренние блоки, карточки, мягкие фоны
    light: '#4B463F',
  },
  champagne: {
    DEFAULT: '#C6AA74', // линии, hover, акценты, номера секций
    deep: '#B99A63',
  },
  burgundy: '#4A1F24',  // редкий акцент (RADAR Races)
}
```

**Правило баланса**: ~80% сайта — graphite / ivory / stone, ~15% — taupe, ~5% — champagne / burgundy. Champagne используется как тонкая линия, номер секции, hover-акцент — **не** как декоративное заполнение.

Никакого ярко-синего, кислотных цветов, градиентов.

### 2.2 Типографика

Три уровня шрифтов, подключить через Google Fonts/Fontsource:

- **Заголовки (display/serif)**: `Playfair Display` или `Cormorant Garamond` — строгий, интеллектуальный editorial-serif. Используется для h1, h2, крупных headline.
- **Основной текст (sans)**: `Manrope` или `Inter` — нейтральный, читаемый, зрелый grotesk.
- **Малые подписи/labels/номера секций (mono)**: `IBM Plex Mono` или `Space Mono` — для «01 / 06», «Practice Director», лейблов.

Заведи в Tailwind как `fontFamily: { display: [...], sans: [...], mono: [...] }`.

Размеры (создать утилитарные классы или typography-компоненты `Heading1`, `Heading2`, `Subheadline`, `Body`, `Label`, `CTAText`):

| Элемент | Desktop | Mobile |
|---|---|---|
| Hero headline | 68–92px | 38–46px |
| Section headline | 44–64px | 30–38px |
| Subheadline | 22–28px | 19–22px |
| Body text | 17–20px | 16–18px |
| Small labels | 11–13px | 11–12px |
| CTA | 13–15px | 13–15px |

Правило: не делать текст мелким «ради элегантности» — премиальный сайт не должен заставлять читателя щуриться.

### 2.3 Сетка и отступы

- Desktop: рабочая ширина контента 1160–1200px, 12-колоночная сетка, боковые поля 40–80px, вертикальные отступы между крупными секциями 140–220px.
- Tablet: ширина контента 90%, отступы между секциями 100–140px.
- Mobile: боковые поля 20–24px, отступы между секциями 80–110px, карточки в один столбец.

Заведи `Container` (max-width + responsive padding) и `Section` (вертикальные отступы + опциональный фон/заголовок/подзаголовок/номер секции) как базовые layout-компоненты — все секции собираются через них.

**Главное правило**: воздух — часть дизайна. Если секция кажется «слишком свободной», это, скорее всего, признак того, что она стала дороже, а не ошибка.

### 2.4 Кнопки / CTA

Использовать **только** эти формулировки CTA по сайту (в зависимости от контекста секции):

- «Обсудить задачу»
- «Получить структуру проекта под ваш запрос»
- «Запросить мастер-класс»
- «Запросить интерактивную лекцию / подкаст»
- «Запросить специальный формат RADAR»
- «Запросить релевантный пример под вашу задачу»
- «Обсудить, какая команда нужна под вашу ситуацию»

**Primary button**: прямоугольная (radius 0–2px), тонкая рамка, без тени, без градиента, без иконки-стрелки (или очень тонкая стрелка). Фон `ivory.DEFAULT` (#F4F0E8), текст `graphite.deep` (#111111). Hover: фон `champagne.DEFAULT` (#C6AA74), текст остаётся `#111111`.

**Secondary button**: прозрачный фон, тонкая рамка (`ivory` или `graphite` в зависимости от секции), текст в тон. Hover — лёгкое заполнение (5–8% по светлоте).

Никогда не использовать: «Купить», «Записаться», «Получить скидку», «Забронировать место», pill-кнопки в стиле SaaS, яркие/кислотные цвета кнопок.

### 2.5 Анимации

Реализовать через CSS transitions/IntersectionObserver или библиотеку типа `framer-motion`, но **очень сдержанно**:

- Плавное появление текста и fade-in карточек: 0.4–0.8 сек, delay между элементами 0.1–0.2 сек, easing мягкий (`ease-out`, без bounce/spring).
- Line reveal — тонкая линия медленно проявляется/растягивается при появлении секции в зоне видимости.
- Hover на карточках: рамка становится чуть светлее (champagne с низкой прозрачностью), появляется тонкая линия, текст CTA смещается на 4–6px вправо, фон меняется максимум на 5–8% по светлоте — без увеличения, без тени, без подпрыгивания.
- Минимальный parallax допустим только на абстрактном radar-элементе в hero.

Запрещено: bounce, spinning icons, counters, агрессивный parallax, быстрые «вылеты», вращение элементов, «вау»-анимации.

## 3. Структура проекта (модульность и контент отдельно от UI)

```
src/
  app/                  # роутинг, layout, провайдеры
  pages/
    HomePage.tsx        # собирает все секции лендинга по порядку
  components/
    ui/                  # Button, Container, Section, SectionLabel (01/06),
                         # ThinLine, Tag, FormField
    layout/              # Header (nav), Footer
    sections/            # один компонент на секцию лендинга (см. раздел 5)
    radar/               # абстрактная radar-графика (SVG: окружности, линии
                         # сканирования, координатная сетка) — переиспользуемые
                         # SVG-компоненты для hero и фоновых акцентов
  content/                # ВСЁ текстовое содержимое, типизировано (см. ниже)
  hooks/
  lib/
  styles/
```

### Контентный слой

Весь текст (заголовки, подзаголовки, тексты, списки, CTA, данные практик и т.д.) хранится в `src/content/*.ts` как типизированные объекты/массивы — компоненты секций только рендерят эти данные. Это даёт возможность редактировать копирайт без правок верстки.

`src/content/types.ts` — общие типы: `CTA`, `SectionMeta { id, number, label }`, `StatementItem`, `SolutionCard`, `DeepDiveSection`, `PracticeCard`, `ProcessStep`, `AdvantageItem`, `ResultItem`, `FormField`, `FooterColumn` и т.д.

Список контентных файлов и их содержимое — раздел 5. Каждый файл должен содержать комментарий вида «чтобы добавить/изменить пункт — отредактируйте массив ниже, типы проверят структуру».

### 5.1 Hero (`sections/Hero.tsx`, `content/hero.ts`)

Кастомная секция (Zero-Block-уровень детализации, не из стандартных блоков).

  > Бутик-партнёрство для сложных управленческих ситуаций

  > RADAR EXEC работает с собственниками, CEO, C-level лидерами и топ-командами, когда системе нужны ясность, собранность и точная архитектура действий.

Подаётся как «аналитическая записка»: слева крупный заголовок, справа список утверждений, каждое отделено тонкой линией, с маленькими номерами 01–08. Без иконок. Фон ivory или очень тёмный графит (контраст с hero).

- Заголовок (H2):

  > Когда прежняя управленческая система больше не держит нагрузку

- Подзаголовок:

  > Сокращения, перераспределённые команды, давление на эффективность и потеря экспертизы меняют саму ткань управления.

- Список утверждений (массив строк, рендерится с номерами 01–08):

  1. Команды стали меньше, а зона ответственности лидеров — шире.
  2. Эффективность всё чаще оценивается не по активности, а по измеримому результату.
  3. Топ-команды должны принимать решения быстрее, но часто делают это при ослабленной координации.
  4. Сильные люди уходят вместе с методологиями, практиками и институциональной памятью.
  5. После реструктуризации лидеры управляют смешанными и заново собранными командами.
  6. Разные поколения работают рядом, но по-разному понимают коммуникацию, автономию, скорость и технологии.
  7. В таких условиях обучение само по себе уже не решает задачу.
  8. Нужна точная работа с управленческой системой.

Самопроверка перед тем как считать секцию готовой — 5 вопросов:

1. Это выглядит как premium advisory, а не как тренинг?
2. Это выглядит спокойно, дорого и точно?
3. Это помогает понять сложность RADAR EXEC, а не упрощает её?
4. Это не похоже на стандартный шаблон?
5. Это усиливает доверие собственника, CEO или C-level лидера?

Если хотя бы на один вопрос ответ «нет» — секцию нужно переделать.
